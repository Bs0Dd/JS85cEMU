Getting started

  1. Turn on the MK. "?" signals the need to initialize the RAM.


  2. Press the "начальная установка" (initial setup) button with a thin (non-conductive) object. The CPU, ROM, RAM and display will be tested.


  3. Next, the system will offer to check the keyboard. You can press various keys to check their functionality. To finish the test, you need to press (in sequence) the keys УПР + EXE.


  4. The encryptor will display a welcome screen. To start the setup, you must enter (not displayed on the screen) the code word "АЗИМУТ" (azimuth).


  5. The system will perform a short test and display the ROM checksum. For all known (at the time of writing) devices, it is 04367402034 (ROM chip code is 061). The Н key will start the test again, the Д key will start the setup procedure.


  6. The system will ask you to set a password of 5 characters. Remember it so that you can unlock the encryptor later! The password is confirmed by the key combination УПР + EXE.


  7. Now you need to enter a DKL (long-term key, LTK) into the system, which is a sequence of 110 digits (divided into 22 blocks of 5 digits). The last two blocks (10 characters) are the checksum of the key (protection against input errors). Such keys were supplied with the device, or were generated by a (lost) program on an IBM PC. At the moment, the key can be generated in this emulator, using the program DKLKM85C or utility for Python by kaseiiro. Input confirmation — УПР + EXE. As an example, the following key is given:

    84534 45986 35465 64750 69746 75562 96281 96471 16889 77629 94879
    96394 73073 45415 29900 39356 54944 10712 85757 23266 32131 18232


  8. The system will prompt you to select the type of data that will be encrypted in the future (this can only be changed when entering a new key). This can be either alphanumeric information or purely digital (the numbers will be displayed separated into blocks).


  9. Now you need to specify the length of the groups into which the numbers will be divided. These can be groups of 2 to 5 digits each (only 2 groups are always shown on the screen). Input confirmation — УПР + EXE.


  10. The encryptor configuration is complete! Now you can use it to encrypt and decrypt data.

Powering on encryptor

  1. Turn on the MK. A greeting will be displayed on the screen. To unlock the encryptor, you must enter the password (the input is not displayed). ATTENTION! After entering 25 incorrect characters (the counter is reset when the encryptor is turned off), the password brute-force protection will be activated and the encryptor will destroy user data in RAM, then display the welcome setup screen. In this case, you need to re-set up the device, starting from point 4.


  2. The system will perform a short test and display the ROM checksum. The Н key will start the test again, the Д key will open the main menu.


  3. The device is now ready to use!

Designation of special keys

     — encryption mode enable key ("Encrypt").

     — decryption mode enable key ("Decrypt").

     — key for switching on the DKL / LTK change mode ("Key").

     — key for deleting the current character (analogous to Delete).

     — input lock key (view and scroll only).

     — keyboard layout change key (Russian and Latin).

     — one-time key generation key (markant).

     — scroll key 12 characters (or two blocks) back.

     — scroll key 12 characters (or two blocks) forward.

     — modifier key for additional functions (control).

Functions of keys with modifier УПР (CTRL)

     + — go to the beginning of the line.

     + — go to end of line.

     + — unlock input.

     + — probably insert mode, not implemented in firmware.

     + — decrypting correction: return to ciphertext input for editing.

     + — decrypting correction: shift gamma back 1 digit.

     + — decrypting correction: skip 1 digit of ciphertext.

     + — confirmation of data entry.

Colored key captions

  • BLUE color captions are entered when using the Latin layout.

  • RED color captions are entered using the УПР key.

Display segment designations


  • EXT — Latin layout is active.

  • S — УПР modifier pressed.

  • F — not used.

  • RUN — text decryption mode.

  • WRT — text encryption mode.

  • DEG — long-term key input mode.

  • RAD — encryption in alphanumeric mode.

  • GRA — encryption in digital mode.

  • TR — input lock is active.

  • STOP — character input limit reached.


  • Seven-segment indicators typically show the number of the current symbol or block.


(alternative version of segments, according to their purpose)

Data encryption mode

  1. To activate the mode in the main menu, use the key .
    Agree to activate it with the Д key.


  2. Before the encryption process begins, it is necessary to generate a random one-time key - a markant. To do this, press the МРК key 10 times.


  3. Now you need to enter the text that will be encrypted (or numbers, if you are using digital encryption mode). The string "ШИФРАТОР ANCRIPT" (ANCRIPT ENCRYPTOR) is given as an example. Confirm with the combination УПР + EXE.


  4. The string is encrypted into a set of numbers, which are divided into blocks of the length specified during setup (regardless of the length of the blocks, only 2 blocks are always displayed on the screen). The ciphertext is ready for transmission. Pressing УПР + EXE returns to the main menu.

    87809 90512 93160 35334 13316 10843 34233 22345 40949

Data decryption mode

  1. To activate the mode in the main menu, use the key .
    Agree to activate it with the Д key.


  2. Now enter the ciphertext blocks obtained during encryption. Confirm the entry by УПР + EXE.


  3. The decrypted string is visible on the screen. If an error was made when entering the blocks and the string was decrypted incorrectly, you can return to entering the ciphertext using the combination УПР + КР. Also, it is possible to eliminate distortions in the ciphertext using the combinations УПР + -1 and УПР + +1, see Correction of distortions during decoding.


  4. When you click УПР + EXE, before returning to the main menu, the program offers to add additional ciphertext blocks. The main purpose of this function is to decrypt a large amount of data. The encryptor can decrypt less information at a time than it can encrypt. This is due to the need to store the ciphertext in RAM so that you can return to editing it using УПР + КР, so there is less free space for decryption. If the ciphertext does not fit into the permissible limit, you need to decrypt and save the current data, and then enter the remaining ciphertext blocks using the add function.

Long-term key replacement mode

  1. To activate the mode in the main menu, use the key .
    Agree to activate it with the Д key.


  2. The system will ask you to set a new password. Enter and confirm with the keys УПР + EXE.


  3. Now you need to enter a new DKL. Enter 110 digits and confirm with the keys УПР + EXE.


  4. Set the encryption mode (Alphanumeric or Digital).


  5. If necessary, you can change the length of the blocks.


  6. Encryptor reconfiguration completed!

Correction of distortions during decoding

    The system used here encrypts data using the gamma method, where a specially developed sequence (gamma) is combined with the data. This method is convenient because it allows encryption and decryption in one operation, combining the original text or ciphertext with the gamma.

Gamma
123456789...
Data
123456789...

    However, when transmitting ciphertext, distortion, loss of part of the data, or its duplication are possible. If distortion during decryption produces, accordingly, distorted but readable data, then with the loss/duplication of just one digit of the ciphertext, a gap occurs between the data and the gamma, which makes it impossible to decrypt the remaining data. The encryptor has the ability to correct such situations. It is important to understand that the first 10 digits of the ciphertext contain a one-time key (markant), the distortion of which immediately leads to the impossibility of decrypting the entire ciphertext, correction will not help here.

Gamma
123456789...
Data
123489101112...
Gamma
123456789...
Data
123454567...

    Let's take the previously obtained ciphertext from the encryption example:

87809 90512 93160 35334 13316 10843 34233 22345 40949

    As you know, the string "ШИФРАТОР ANCRIPT" (ANCRIPT ENCRYPTOR) is encrypted here. Now imagine that three ciphertext digits were lost during transmission:

87809 90512 93160 35334 13316 10843 34233 22345 40949

    Let's try to enter the existing ciphertext into the encryptor:


    Due to the loss, the text after the 6th letter became completely unreadable. The key combinations УПР + -1 or +1 are intended for correction. The -1 key shifts the gamma back by 1 digit, the +1 key skips one digit of data. In this case, since 3 digits are lost, the gamma needs to be shifted back by the same number of digits.

Gamma
101112131415161718...
Data
101112131617181920...
Gamma
101112131617181920...
Data
101112131617181920...

    Let's move the cursor to the position where the distortion began (in digital mode, for some reason, the cursor only moves by blocks, however, the correction itself does not make much sense here). Press the combination УПР + -1 three times. And indeed, we got a quite readable string "ШИФРАТ ANCRIPT", only 2 letters were lost in the text (each symbol in alphanumeric mode is encoded by 2 digits).


    Now let's imagine that during transmission there was a duplication of 4 digits:

87809 90512 93160 3530 35334 13316 10843 34233 22345 40949

    After just 4 letters the decoding was broken.


    For normalization it is necessary to skip 4 duplicated digits:

Gamma
5678910111213...
Data
567856789...
Gamma
5678910111213...
Data
5678910111213...

    Place the cursor on the position and press УПР + +1 4 times. The data is completely decrypted now.

Device information

  • Plaintext size: 750 characters or 1500 numeric characters

  • Volume of information decrypted at one time:

    • Alphanumeric mode: 535/536* characters

    • Digital mode: 800/802/803/804** digits

  • The volume of information encrypted on one long-term key: up to 3,000,000 characters

  • Long-term key: 10100 variants

  • One-time key: 1010 variants

  • Encryption algorithm: nonlinear algorithm of the highest complexity "Angstrem-3"

  • Encoding: similar to KOI-7, with a shift of 32 positions.

* For blocks of 2 digits, you can decrypt 1 more symbol.
** For blocks of 5/4/3/2 digits.
